## 1. 실험 목적

본 과제의 목적은 PyTorch, TensorFlow 같은 딥러닝 프레임워크를 사용하지 않고 NumPy만으로 MNIST 손글씨 숫자 분류 신경망을 구현하는 것이다.

현재 구현은 Affine 계층, ReLU, Softmax, Cross Entropy Loss, Batch Normalization, Dropout, Adam optimizer, 학습 루프, 평가 함수까지 직접 작성한 구조이다. `evaluate(model, x, y)`를 통해 테스트 정확도와 총 파라미터 수를 확인하며, 목표 정확도는 95% 이상이다.

---

## 2. 모델 구조

현재 `src/network.py`의 `NeuralNetwork` 구현 기준 모델 구조는 다음과 같다.

| 항목 | 내용 |
| --- | --- |
| 입력층 | 784차원 입력, 28x28 MNIST 이미지를 펼친 벡터 |
| 은닉층 수 | 2개 |
| 은닉층 차원 | 512, 256 |
| 출력층 | 10차원 출력, 숫자 0~9 분류 |
| 활성화 함수 | ReLU |
| BatchNorm | `use_batchnorm=True`일 때 각 은닉층 뒤에 적용 |
| Dropout | `use_dropout=True`일 때 각 은닉층 뒤에 적용 |
| Dropout 비율 | 기본값 `0.5` |
| 출력 처리 | 마지막 Affine 출력 뒤 Softmax 적용 |

```text
Input(784)
-> Affine(512) -> BatchNorm -> ReLU -> Dropout(0.5)
-> Affine(256) -> BatchNorm -> ReLU -> Dropout(0.5)
-> Affine(10)
-> Softmax
```

`predict()`에서는 `train=False`로 forward를 수행하여 BatchNorm과 Dropout이 추론 모드로 동작한다.

### Learning rate 비교 결과

다섯 실험은 모델 구조와 총 파라미터 수가 동일하므로, 차이는 learning rate 설정에 따른 학습 양상으로 볼 수 있다. `lr=10`은 loss가 3.0929에서 2.8731로 7.1% 감소했지만, 중간 epoch에서 3.2291까지 다시 상승하는 등 진동이 컸고 test accuracy도 9.80%에 머물렀다. MNIST는 10개 클래스를 분류하므로 9.80%는 무작위 추측 수준에 가깝다. 따라서 `lr=10`은 learning rate가 지나치게 커 안정적으로 학습하지 못한 설정으로 해석된다.

`lr=0.1`은 test accuracy 97.69%로 목표 정확도 95%를 넘었지만, 최종 loss가 0.1900으로 다른 정상 학습 설정보다 높고 후반부 감소도 둔화되었다. `lr=0.01`은 최종 loss 0.0572, test accuracy 98.33%로 안정적인 결과를 보였고, `lr=0.001`은 최종 loss 0.0445, test accuracy 98.46%로 더 좋은 성능을 보였다.

`lr=0.0016`은 최종 train loss가 0.0413으로 가장 낮고, test accuracy도 98.53%로 가장 높았다. 따라서 이번 실험에서 최종 모델의 learning rate로는 `lr=0.0016`을 선택하는 것이 가장 적절하다. 다만 최종 loss가 더 낮다고 해서 test accuracy가 항상 더 높아지는 것은 아니다. Loss는 정답 클래스에 부여한 확률의 크기까지 반영하지만, accuracy는 최종 예측 클래스가 정답과 같은지만 판단한다. 이번 실험에서는 `lr=0.0016`이 두 지표 모두에서 가장 좋게 나타났기 때문에 최종 설정으로 선택했다.

## 6. 회고

현재 구현은 NumPy만 사용해 신경망의 핵심 구성 요소를 직접 작성했다는 점에서 과제 요구사항에 맞는다. Affine 계층의 forward/backward, BatchNorm의 학습/추론 모드, Dropout의 학습/추론 모드, Softmax와 Cross Entropy 기반 gradient 흐름, Adam optimizer의 파라미터 업데이트 과정을 코드로 확인할 수 있다.

모델은 `784 -> 512 -> 256 -> 10` 구조로 구성되어 있으며, BatchNorm을 통해 학습 안정성을 높이고 Dropout을 통해 과적합을 줄이도록 설계했다. Learning rate 비교 결과 `lr=10`은 loss가 불안정하게 진동하고 test accuracy가 9.80%에 그쳐 학습에 실패한 설정으로 볼 수 있었다. 반면 `lr=0.0016`은 최종 train loss 0.0413과 test accuracy 98.53%로 가장 좋은 결과를 보였다. 이번 비교를 통해 너무 큰 learning rate는 학습을 불안정하게 만들 수 있고, 최종 loss와 test accuracy는 관련이 있지만 반드시 같은 순서로 좋아지는 지표는 아니라는 점을 확인했다. 최종 모델에는 두 지표가 모두 가장 우수한 `lr=0.0016`을 적용하는 것이 적절하다고 판단했다.
